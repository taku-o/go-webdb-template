# GitHub Pages対応の要件定義書

## 1. 概要

### 1.1 プロジェクト情報
- **プロジェクト名**: go-webdb-template
- **Feature名**: 0066-githubpages
- **作成日**: 2026-01-27
- **関連Issue**: https://github.com/taku-o/go-webdb-template/issues/136

### 1.2 目的
プロジェクトのGitHub Pagesページを用意し、プロジェクトの概要・目的、セットアップ手順、各種URL情報を公開する。多言語対応（日本語・英語）を実現し、プロジェクトの理解と導入を容易にする。

### 1.3 スコープ
- GitHub Pages用のMarkdownファイルの作成
- 多言語対応（日本語・英語）のページ構成
- プロジェクト概要・目的の記載
- クライアントサーバーを動作させるまでの詳細なセットアップ手順の記載
- 各種URL情報（クライアントURL、APIサーバーdoc URL、AdminサーバーURL、認証情報）の記載
- デザインの実装（それなりに良いデザイン）

**本実装の範囲外**:
- GitHub Pagesの自動デプロイ設定（GitHub Actions等）
- 既存のdocsディレクトリの変更
- プロジェクトの機能追加・変更

## 2. 背景・現状分析

### 2.1 現状
- プロジェクトには詳細なドキュメント（README.md、docs/配下の各種ドキュメント）が存在する
- セットアップ手順はREADME.mdとdocs/Initial-Setup.mdに記載されている
- GitHub Pagesは未設定
- プロジェクトの概要やセットアップ手順を外部に公開する仕組みがない

### 2.2 必要性
- プロジェクトの概要・目的を外部に公開する
- セットアップ手順を分かりやすく公開する
- 多言語対応により、日本語・英語の両方のユーザーに対応する
- 各種URL情報を一箇所にまとめて公開する

### 2.3 実現可否
- GitHub PagesはMarkdownファイルをサポートしている
- Jekyllを使用することで、多言語対応やテーマの適用が可能
- 既存のdocsディレクトリの内容を活用できる

## 3. 機能要件

### 3.1 ページ構成

#### 3.1.1 ディレクトリ構造
以下のディレクトリ構造でGitHub Pages用のMarkdownファイルを配置する：

```
docs/
├── index.md              # トップページ（言語選択）
├── en/                   # 英語版ページ
│   ├── index.md         # 英語版トップページ
│   ├── about.md         # プロジェクト概要・目的
│   └── setup.md         # セットアップ手順
└── ja/                   # 日本語版ページ
    ├── index.md         # 日本語版トップページ
    ├── about.md         # プロジェクト概要・目的
    └── setup.md         # セットアップ手順
```

#### 3.1.2 トップページ（docs/index.md）
- 言語選択ページとして機能
- 日本語版と英語版へのリンクを提供
- シンプルで分かりやすいデザイン

#### 3.1.3 各言語版のトップページ（docs/ja/index.md、docs/en/index.md）
- プロジェクト名と簡単な説明
- 主要なページへのナビゲーションリンク
  - About（プロジェクト概要・目的）
  - Setup（セットアップ手順）
- 言語切り替えリンク

#### 3.1.4 Aboutページ（docs/ja/about.md、docs/en/about.md）
- プロジェクトの概要・目的
- 技術スタックの概要
- 主要な機能の紹介
- README.mdの「プロジェクト概要」「特徴」セクションの内容を参考にする

#### 3.1.5 Setupページ（docs/ja/setup.md、docs/en/setup.md）
- クライアントサーバーを動作させるまでの詳細なセットアップ手順
- 以下の内容を含む：
  - docs/Initial-Setup.mdの内容（前提条件、Homebrew、Node、Atlas、uv等のインストール手順）
  - Redis起動手順
  - PostgreSQL起動手順
  - データベースの初期化手順（マイグレーション適用）
  - Auth0アカウントの設定手順
  - client/.env.localの用意手順
  - APIサーバー起動手順
  - クライアントサーバー起動手順
  - Adminサーバー起動手順
  - 各種URL情報
    - クライアントURL: http://localhost:3000
    - APIサーバーdoc URL: http://localhost:8080/docs
    - AdminサーバーURL: http://localhost:8081/admin
      - ID: admin
      - Password: admin123

### 3.2 デザイン要件

#### 3.2.1 全体的なデザイン
- シンプルで読みやすいデザイン
- モダンな見た目
- 適切な余白とタイポグラフィ
- コードブロックの適切な表示

#### 3.2.2 ナビゲーション
- 各ページ間の移動が容易
- 言語切り替えが簡単
- トップページへの戻りが容易

#### 3.2.3 レスポンシブデザイン
- モバイルデバイスでも読みやすい
- タブレットでも適切に表示される

### 3.3 コンテンツ要件

#### 3.3.1 日本語版
- すべてのコンテンツを日本語で記載
- 技術用語は適切に日本語化または英語表記を併記

#### 3.3.2 英語版
- すべてのコンテンツを英語で記載
- 技術用語は適切に英語表記

#### 3.3.3 コンテンツの整合性
- 日本語版と英語版の内容は一致させる
- 既存のREADME.mdやdocs/配下のドキュメントと整合性を保つ

## 4. 非機能要件

### 4.1 可用性
- GitHub Pagesが正常に動作している限り、ページにアクセス可能
- ページの読み込み速度は適切な範囲内

### 4.2 保守性
- Markdownファイルで管理されるため、編集が容易
- 既存のdocsディレクトリと整合性を保つ
- 将来的な拡張が容易

### 4.3 互換性
- GitHub Pagesの標準機能（Jekyll）を使用
- 標準的なMarkdown記法を使用
- 特殊なプラグインや拡張機能は使用しない（標準機能のみ）

## 5. 制約事項

### 5.1 技術的制約
- GitHub PagesはJekyllを使用（デフォルト）
- Markdownファイルのみを使用（HTMLの直接記述は最小限）
- 既存のdocsディレクトリの構造を維持

### 5.2 実装上の制約
- 既存のdocsディレクトリ内のファイルは変更しない
- 新しいファイルのみを追加する
- GitHub Pagesの設定は別途必要（本実装の範囲外）

### 5.3 コンテンツの制約
- 既存のREADME.mdやdocs/配下のドキュメントの内容を参考にする
- 記載されていない情報は追加しない
- 将来導入予定の機能は考慮しない

## 8. 実装上の注意事項

### 8.1 ファイル配置
- GitHub Pagesは`docs/`ディレクトリをソースとして使用できる
- 既存のdocsディレクトリ内のファイルと競合しないように注意する
- 新しいファイルのみを追加する

### 8.2 コンテンツの作成
- 既存のREADME.mdやdocs/配下のドキュメントの内容を参考にする
- 記載されていない情報は追加しない
- 将来導入予定の機能は考慮しない

### 8.3 デザインの実装
- Jekyllの標準機能を使用する
- 必要に応じて`_config.yml`を作成する（Jekyll設定）
- テーマはGitHub Pagesのデフォルトテーマまたはminimaテーマを使用する

### 8.4 多言語対応
- 日本語版と英語版の内容は一致させる
- 技術用語は適切に翻訳または英語表記を併記する

### 8.5 GitHub Pages設定
- GitHub Pagesの設定は別途必要（本実装の範囲外）
- 設定方法は別途ドキュメント化するか、ユーザーに案内する
