# タスク完了時のチェックリスト

## 開発完了時の確認事項

### 1. テストの実行
```bash
cd server
go test ./... -v
```
- 全テストがパスすることを確認

### 2. ビルド確認
```bash
cd server
go build ./cmd/server/...
go build ./cmd/admin/...
```
- ビルドエラーがないことを確認

### 3. 動作確認
- サーバーを起動して動作を確認
- エラーが発生しないことを確認

### 4. コード品質
- 不要なコードは削除
- コメントは必要な箇所のみ
- エラーハンドリングが適切

### 5. ドキュメント確認
- `docs/` 配下のドキュメントと整合性があること
- 仕様（`.kiro/specs/`）と一致していること

## CLAUDE.local.mdのルール（重要）

- ドキュメントにない余計な機能は作らない
- 将来導入予定の機能は考慮しない
- 最適化やフォールバック機能は不要
- テストコードの変更はユーザー許可が必要
- 一括処理スクリプトの使用はユーザー許可が必要
- 設計と実装が一致しない場合はユーザーに報告
